Siempre depende el lenguaje que escojan, ya que su entorno de desarrollo va a cambiar un poco, quizás otros van a necesitar instalar sus propias herramientas como: copiladores, interpretes, etc. 



# Bases

es importante sobre sobre las variables, funciones, objetos, listas, clases, condicionales, loops, packages/Modules. La mayoría de los lenguajes requieren de estos y por esa razón es exencial saber sobre estos. 


# Servidor Web 

Para este es necesario conocer el protocolo HTTP, todas las aplicaciones que utilizamos están por encima de este protocolo, este es un protocolo de la web, donde cualquier cosa que se cree necesitar conocer sus reglas como: 

* Métodos 
* Peticiones 
* Respuestas 
* Código de estatus
* Cabeceras 
* Cookies 

# Server

Para que el lenguaje pueda comunicarse y funcionar como un servidor, se utilizan paquetes o módulos internos que permiten manejar

* Net: Comunicación a través de redes.
* Files: Manejo de archivos dentro del sistema.
* HTTP: Soporte nativo para el protocolo.
* CLI: Creación de aplicaciones de consola. 

# Herramientas y Conceptos Adicionales

* Clientes REST: Aplicaciones como Postman, Insomnia o extensiones como Thunder Client para probar tus rutas.


* Frameworks: Herramientas que automatizan tareas repetitivas (ej. Express para Node, Django para Python, Laravelpara PHP).


* Arquitecturas de API: Diferentes formas de responder datos, como REST (JSON), SOAP (XML), GraphQL o gRPC.

* Bases de Datos: Es vital aprender SQL (PostgreSQL es muy recomendado) y entender las bases NoSQL como MongoDB. También el uso de ORMs para interactuar con ellas desde el código.


* Seguridad: Conocer el OWASP Top 10, autenticación con JWT (JSON Web Tokens) y métodos como 2FA u OTP.

* Despliegue y Nube: Entender conceptos de PaaS (Heroku, Render) e IaaS (AWS, Azure, Google Cloud), además del uso de Docker para contenedores.
